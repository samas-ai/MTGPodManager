## 📂 Architectural Decisions
*(Log specific choices made during the build here so future agents respect them)*
- 2026-06-11 — **Project scope: strictly NON-COMMERCIAL, for-fun (user-confirmed).** Never monetize without revisiting this. This clears the Phase 11 D0 legal gate: WotC mana symbols + card art are permitted under the **Fan Content Policy** *because* the app is non-commercial — requires the Fan Content Policy footer disclaimer ("unofficial Fan Content… not approved/endorsed by Wizards"), and Scryfall imagery requires **artist attribution** (no implied endorsement, cache respectfully). If monetization is ever proposed, the entire D1–D5 theming track must be re-scoped to drop WotC-derived IP.
- 2026-06-11 — **Phase 10 (table-side UX).** Nav is a client BottomNav rendered in the root layout, path-gated (hidden on `/`, `/sign-in`, `/sign-up`, `/match`, `/join`) — no route-group migration. Stats is NOT a tab (group-scoped). Dark mode: reuse existing `.dark` tokens via a no-flash inline `<script>` in `<body>` (CSP allows `'unsafe-inline'` scripts) + `suppressHydrationWarning` on `<html>`; Profile has the System/Light/Dark toggle. QR via `qrcode` lib → server-rendered inline SVG (CSP-safe, fixed black-on-white since a barcode is functional, not themeable); `getOrigin()` from forwarding headers. Invite QR → `/join/[code]` landing; auth actions thread a `next` param (safePath-guarded) carried across sign-in/up + the email-confirm branch. Table Mode is local-only React state (no sync, like the normal counter); Wake Lock typed via a local interface (not in all TS DOM libs). B6 rematch = new open match in the same group (no participant pre-seeding — re-verify per-match). B5 group mgmt: all writes via SECURITY DEFINER RPCs (group_members still has no write policy); `is_group_admin()` helper; last-admin guards on leave/remove/demote prevent orphaning. **pgTAP lesson:** assertions that read a table after a member leaves must run as `postgres`, not the departed member (RLS correctly hides the roster from non-members).
- 2026-06-10 — **Phase 9 (P1 stats).** Placement: nullable smallint on `match_participants`, RPC-only writes (`mp_*_self` WITH CHECK `placement is null`), `finalize_match(uuid,uuid,jsonb default null)` validates strict 1..N permutation with winner first; winner-only mode unchanged; winners backfilled to 1. Deck stats: `group_deck_winrates` + `group_head_to_head` (directional pairs; client keeps `player_id < opponent_id`) as `security_invoker` views. Standings-over-time: pure `winRateSeries` fold + dependency-free SVG `Sparkline` (computed on read, no migration). Notes/tags: host-while-open via existing `matches_update_open`, freeze at finalize; `valid_match_tags` immutable fn as CHECK backstop. E2E: golden-path spec asserts the live join-status update without reload; CI runs it against the local stack (polling transport for determinism). zod-4 quirk: `uuid` format is strict RFC 4122 — fixture ids must be valid v4 (and hex-only — no `p`/`q` typos).
- 2026-06-10 — **Phase 7 Pass 1: design restyle + a11y (token-based, no behavior change).** "Arcane parchment" theme in `globals.css` (AA-contrast palette + `--mtg-w/u/b/r/g` mana tokens + `prefers-reduced-motion`); fonts via `next/font/google` (Cinzel display + Inter body) wired as `--font-display`/`--font-sans` in `layout.tsx` (also skip-to-content link + `#main`). New primitives: `ui/badge`, `ui/skeleton`, `ui/page-header`, `features/color-pips` (WUBRG pips pair color+letter, aria-labelled). Route states: `loading.tsx` (groups, groups/[id], stats/[id], profile/decks) + `error.tsx`/`not-found.tsx`/`global-error.tsx`. `CardTitle`/headings use display font; badges for admin/verified/imported; life-counter buttons aria-labelled; per-page `metadata.title`. Tokens-only (no raw hex in components) so a dark-mode toggle later is just a token swap.
- 2026-06-09 — **Supabase BaaS with RLS as the authorization boundary** (not a separate API service). Keep secrets/import/finalize logic strictly server-side; RLS is the net under that discipline.
- 2026-06-09 — **Match finalization lives in a `SECURITY DEFINER` Postgres function** `finalize_match(match_id, winner_id)`, not in app-layer code. An RLS `with check` forbids plain updates from flipping `status` to `finalized`; the function is the only door. Atomic, row-locked.
- 2026-06-09 — **Result model: winner-only** (`matches.winner_user_id`, validated to be a participant). Full finishing order deferred (clean future migration: add `placement` to `match_participants`).
- 2026-06-09 — **Live join-status transport: Realtime Postgres Changes** on `match_participants` filtered by `match_id`, with a **polling fallback already written** to degrade gracefully if free-tier Realtime limits bite. The DB is the single source of truth for participation.
- 2026-06-09 — **Life totals are local-only** (host screen React state). Synced life across devices is explicitly out of scope; keeps Realtime payloads tiny.
- 2026-06-09 — **Archidekt import is best-effort, server-side only**, with a manual two-field fallback (name + Scryfall-resolved commander). Deck *identity* (name + commander + color identity) is decoupled from full card list; `card_data` is nullable. No MVP stat depends on the card list.
- 2026-06-09 — **Snapshot deck identity onto `match_participants`** (`deck_name_snapshot`, `commander_snapshot`) so history stays correct after renames/deletes and `decks` can stay strictly owner-private.
- 2026-06-09 — **Stats via `security_invoker = on` views** (`group_player_winrates`, `group_deck_play_counts`) — group-scoped automatically via base-table RLS; recompute on read (fine at pod scale).
- 2026-06-09 — **Stack pinned (Phase 1 scaffold):** Next.js **14.2.35** (App Router, patched), React 18, `@supabase/ssr` **0.12** + `@supabase/supabase-js` **2.108** (must stay version-aligned — see quirk), Tailwind 3 + design-token CSS vars, shadcn-style UI primitives (copied-in, no Radix), Zod, Vitest + Playwright. No global state lib.

- 2026-06-09 — **F1 membership RPCs** `create_group(p_name)`, `create_invite(p_group_id)`, `accept_invite(p_code)` are all `SECURITY DEFINER`; `group_members` has **no insert policy**, so RPCs are the only write path. - 2026-06-10 — **pgTAP fixture pattern (learned from 3 CI iterations):** suites that define `tests.act_as()` need `create schema if not exists tests; grant usage on schema tests to authenticated;` AND `grant select on <temp_table> to authenticated;` for any temp fixture table read while acting as a user (temp tables are owned by postgres). - 2026-06-10 — **GitHub Actions Node 20 deprecation (deadline 2026-06-16):** CI log warns `actions/checkout@v4`, `actions/setup-node@v4`, `supabase/setup-cli@v1` run on Node 20, which gets force-switched to Node 24 on June 16 and removed Sept 16.
